# Исправление проблемы с set_variable для maxRecords=1

## Проблема решена

Добавлена обработка ошибки "maximum number of records is reached" на уровне внешнего catch блока. При возникновении этой ошибки код автоматически использует `setVariableField` как fallback для обновления переменной.

## Реализация

1. **Проверка в основном блоке**: Код проверяет `maxRecords == 1` или `isSingleRecord()` и использует `setVariableField` для таких переменных.

2. **Fallback обработка**: Если проверка не сработала и возникла ошибка "maximum number of records is reached", код автоматически перехватывает ошибку и использует `setVariableField` для обновления переменной.

3. **Двойная защита**: Код имеет два уровня защиты:
   - Проактивная проверка перед вызовом `setVariable`
   - Реактивная обработка ошибки после вызова `setVariable`

## Результат

Проблема решена - код больше не будет натыкаться на ошибку "maximum number of records is reached" для переменных с `maxRecords=1`.
