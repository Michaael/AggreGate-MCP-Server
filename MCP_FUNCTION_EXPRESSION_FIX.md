# Исправление логики создания Expression функций

## Проблема

При создании Expression функций с автоматической генерацией выражения возникала ошибка:
- Сгенерированное выражение: `table("<<result><E>>>",{arg1}+{arg2})`
- Правильное выражение: `table("<<result><E>>",{arg1}+{arg2})`

## Анализ реальной функции test

Из функции `test` в модели `users.admin.models.test_mcp_extended`:
- **expression**: `table("<<result><E>>",{arg1}+{arg2})`
- **type**: 1 (Expression)
- **inputformat**: `<<arg1><E><arg2><E>>`
- **outputformat**: `<<result><E>>`

## Исправления

1. **Автоматическая генерация выражения**:
   - Если выражение не предоставлено, автоматически генерируется на основе `inputFormat` и `outputFormat`
   - Формат: `table("<<outputFormat>>",{field1}+{field2}+...)`

2. **Правильный синтаксис**:
   - Использовать `{arg1}`, `{arg2}` вместо `cell(parameters, "arg1")`
   - Использовать функцию `table()` для создания результата
   - Формат результата: `"<<result><E>>"` (без лишних `>`)

3. **Исправление генерации формата**:
   - Убрать лишний `>` в конце формата
   - Правильный формат: `<<result><E>>` (не `<<result><E>>>`)
