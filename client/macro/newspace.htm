Now that you can see this window, please close the $R{aggregateClient}. Note: You can also print this tutorial. It might be easier to follow it using a piece of paper.

You're now going to pretend you're a regular user, not a system administrator. This part will show you how a regular user can connect to a $R{aggregateServer} using AggreGate Client and a server account previously configured by an Administrator (remember what you did in the previous tutorial? We hope you kept the username and password for that account in a safe place...).

So now, let's say you're User Guy. Follow these steps:

  • Run the $R{aggregateClient}.
  • Enter a username in the User Name textbox. This can be any username -- doesn't have to match the username of the account you have created on the server. This will create a new workspace on the local Client installation. It doesn't log you in to any server just yet.

  • Enter a password in the Password textbox. You're now selecting a new password -- you didn't configure it yet. This will be the password for this user on this particular Client installation. Once you selected it, remember it or write it down somewhere. It lets you access this local workspace. This doesn't have to be the same password used for this user's account on the Server (the password you configured before, as an Administrator). It can be different.
  • You should now get a dialog alerting you that the workspace does not exist:

  • Click Yes to create a new workspace.
  • Voila! Here's your new workspace. Notice the empty System Tree -- no Server connections just yet.

It is now time to create a new Server Account -- connect to your $R{aggregateServer} again, only this time as a regular user (not an Administrator).

To do this, click Help > Open Interactive Guide and work through the tutorial called Create a new Server Account.

You've done this tutorial before, for another workspace, as an Administrator. Now you should do it from the viewpoint of a regular user, and use the credentials you have previously created for a regular user.

NOTE: Remember to use the user name and password you created for the USER, not the administrator.

You can close this window now, and proceed with the Create a new Server Account interactive tutorial.
